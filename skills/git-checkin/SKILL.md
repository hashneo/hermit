# git-checkin

## Core principles

- Run `ci-precheck-commit` before staging anything
- Explicit `git add <file>` only — never `git add .` or `git add -A`
- Never commit to `main`
- Work is not complete until a PR is opened via `pr-lifecycle`

---

## Step 0a — Branch safety gate (MANDATORY)

```bash
CURRENT_BRANCH=$(git branch --show-current)

# Refuse work on main
if [ "$CURRENT_BRANCH" = "main" ]; then
  echo "ERROR: cannot commit to main. Create a feature branch first."
  exit 1
fi

# Verify gh CLI is available
command -v gh >/dev/null 2>&1 || { echo "ERROR: gh CLI required"; exit 1; }

# Check current PR state (if a PR exists for this branch)
PR_STATE=$(gh pr view --json state --jq '.state' 2>/dev/null || echo "NONE")
if [ "$PR_STATE" = "MERGED" ] || [ "$PR_STATE" = "CLOSED" ]; then
  echo "ERROR: PR for branch '$CURRENT_BRANCH' is $PR_STATE."
  echo "Create a new branch from main: git checkout main && git pull && git checkout -b feat/my-new-work"
  exit 1
fi

# Show what's ahead of main for coherence check
echo "Commits ahead of main:"
git log --oneline origin/main..HEAD
```

## Step 0b — Confirm feature branch

If no feature branch exists:
```bash
git checkout main && git pull origin main
git checkout -b feat/<descriptive-name>
```

## Step 0.5 — Run ci-precheck-commit

Run the `ci-precheck-commit` skill. Do not proceed if any check fails.

## Step 1 — Review changes

```bash
git status
git diff <file>   # review each changed file before staging
```

## Step 2 — Stage explicitly

```bash
git add path/to/file1 path/to/file2
# Never: git add . or git add -A
# Never stage: .beads/, scripts/__pycache__/, hermit-native/build/, HermitNative.app/
```

## Step 3 — Prepare commit message

Capture a unique temp filename:
```bash
echo /tmp/hermit-commit-msg-$$.txt
```

## Step 4 — Write commit message to temp file

Use the Write file tool (not a heredoc). Follow conventional commits:

```
<type>(<scope>): <short summary>

## Changes
- <what changed and why>
- <key implementation details>

## Testing
- <what was tested>
- <test results>

## Notes
- <breaking changes, follow-ups, caveats>

Generated by <Platform>
Co-Authored-By: Claude <Model-Name> <noreply@anthropic.com>
```

**Types:** `feat`, `fix`, `refactor`, `perf`, `ci`, `docs`, `test`, `chore`

**Scopes (examples):** `rfc`, `accounts`, `keychain`, `dashboard`, `observability`, `native`, `go`, `ci`

> Read the model name from the system context — do NOT hardcode it.

## Step 5 — Commit

```bash
git commit -F /tmp/hermit-commit-msg-<PID>.txt
```

## Step 6 — Push

```bash
# First push (set upstream):
git push -u origin $(git branch --show-current)

# Subsequent pushes:
git push origin $(git branch --show-current)
```

## Step 7 — Clean up

```bash
rm /tmp/hermit-commit-msg-<PID>.txt
```

## Step 8 — Open PR via pr-lifecycle

Work is **not complete** until a PR is opened. Invoke `pr-lifecycle`.
